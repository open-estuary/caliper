The method of Caliper score calculation

We begin the computation by obtaining the original values from the output log which is generated by executing each test tool on target. These original values are then categorized in to Two top Dimensions,Performance and Functional. This is further sub-divide based on which sub-system(CPU, Memory, Algorithm etc) they belong to.

After the classification is done we normalize these original values into a smaller dynamic range. Care is taken to convert all values which are interpreted as "Smaller the better" to "Higher the better" to get a uniform method of interpreting the results as Higher the score, better the performance!.

Total scores for each sub-system are then computed by taking the geometrical mean of all the individual test scenarios which comes under it. This is further calculated by taking the geometrical mean of each set of participating test cases that come under each scenarios

These calculated scores are then compared across platforms and then converted them to percentiles. Which means the highest value in each category will have the value of 100 and rest will be relative percentile score to this. Percentile calculation is applied to each sub system and also for each test cases under specific category and these values are referred in the report.
